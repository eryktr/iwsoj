users.api module
----------------
| `api.py source code <https://github.com/eryktr/iwsoj/blob/master/iwsoj/users/api.py>`_
|
| In file api.py we can find two classes: **CreateUserView** and **AuthUserToken**. Class **CreateUserView** collaborates with class **UserSerializer** from file **user_serializer.py** from directory **serializers**.
|
| Class **UserSerializer** is used to register by the method **create** and save a new user. 
|
| Class **CreateUserView** is used to create a view of the new user of out program. 
|
| Class **AuthUserToken** is used to create an authorization token which is used to get access to available tasks and submissions by the method Bearer.
|
| Bearer is an authentication method which involves security tokens. The tokens are crypted strings generated by the server in response to a login request. If you are interested more about Bearer method, `click here <https://swagger.io/docs/specification/authentication/bearer-authentication/>`_ .
|
| Now it's time to say something about **permission_classes**.
|
| Permission is a functionality of system what determines which request will be granted or denied. We use **AllowAny** and that permission class will allow unrestricted access, even if request is unauthenticated. If you are interested more about permissions and permission classes, `let's see <https://www.django-rest-framework.org/api-guide/permissions/>`_ .
